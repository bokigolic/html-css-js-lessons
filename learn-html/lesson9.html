Lesson 9: Semantic HTML

🧠 What is Semantic HTML?

Semantic HTML uses meaningful tags to describe content. Instead of using <div> for everything, semantic tags like <header> and <article> help browsers and developers understand your content structure.

Why it matters: Semantic HTML improves SEO, accessibility, and code clarity.

📦 Key Semantic Tags

  • <header> – top of a page or section
  • <footer> – bottom of a page or section
  • <main> – the main content (unique per page)
  • <section> – a thematic grouping of content
  • <article> – self-contained, reusable piece (like blog post)
  • <aside> – sidebars, ads, or tangential info
  • <nav> – site or page navigation links

🖼️ Example: Page Layout

<header>
  <h1>My Blog</h1>
</header>

<nav>
  <a href="#">Home</a>
  <a href="#">Posts</a>
</nav>

<main>
  <article>
    <h2>Learning HTML</h2>
    <p>HTML is the structure of the web.</p>
  </article>

  <aside>
    <h3>Related Links</h3>
    <ul>
      <li><a href="#">CSS Basics</a></li>
    </ul>
  </aside>
</main>

<footer>
  <p>Copyright 2025</p>
</footer>

✅ Benefits of Using Semantic Tags

  • Improves SEO rankings and indexing
  • Helps screen readers navigate pages
  • Makes code easier to read and maintain
  • Provides clear structure and intent

🧪 Practice Challenge

Create a simple blog post layout using semantic elements:

  • <header> with title
  • <nav> with 2 links
  • <main> with an <article>
  • <aside> for tips
  • <footer> with copyright

📌 Summary

  • Semantic HTML = meaningful tags
  • Use <main>, <section>, <article>, etc.
  • Improves SEO, accessibility, and structure
