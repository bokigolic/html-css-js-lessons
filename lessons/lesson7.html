📊 Why Use Tables?

Tables are perfect for structured data — schedules, product comparisons, or pricing plans. They help users understand information fast and clearly.

💡 Tip: Use tables for data, not layout. Layout should be handled with CSS flexbox or grid.

🔹 Basic Table Structure

<table>
  <thead>
    <tr>
      <th scope="col">Name</th>
      <th scope="col">Age</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Ana</td>
      <td>28</td>
    </tr>
  </tbody>
</table>

📎 Key Tags: <thead> for headers, <tbody> for data rows, scope="col" for accessibility.

🧩 colspan & rowspan

Use these attributes to merge cells across columns or rows:

<td colspan="2">Merged cell</td>
<td rowspan="2">Spans 2 rows</td>
⚠️ Pro Tip: Always match colspan/rowspan to the table structure to avoid broken layouts.

🎨 Styling Tables with CSS

table {
  border-collapse: collapse;
  width: 100%;
}
th, td {
  border: 1px solid var(--shadow);
  padding: 0.75rem;
  text-align: left;
}
tr:nth-child(even) {
  background-color: var(--row-alt);
}

Add zebra striping, spacing, and hover effects for better UX.

🧪 Example: Weekly Schedule

<table>
  <caption>My Weekly Routine</caption>
  <tr><th>Day</th><th>Activity</th></tr>
  <tr><td>Monday</td><td>Workout</td></tr>
  <tr><td>Tuesday</td><td>Coding Practice</td></tr>
</table>

🧠 Practice Challenge

Create a pricing table with 3 plans. Include:

  • Plan Name
  • Price
  • Features
<table>
  <tr>
    <th>Plan</th>
    <th>Price</th>
    <th>Features</th>
  </tr>
  <tr>
    <td>Basic</td>
    <td>$9/mo</td>
    <td>Email support</td>
  </tr>
  <tr>
    <td>Pro</td>
    <td>$19/mo</td>
    <td>Priority + Email support</td>
  </tr>
</table>

📌 Summary

  • <table> for data structure
  • <thead>, <tbody> for clarity
  • colspan, rowspan for merged cells
  • Use scope and caption for accessibility
  • Style with borders, padding, striping
