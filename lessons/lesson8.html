📨 What Are Forms?

Forms are used to collect user input — from login info to survey responses. They are the main way users interact with your backend.

💡 Accessibility Tip: Always use <label> linked to inputs for better screen reader support.

🧾 Basic Form Structure

<form action="/submit" method="post">
  <label for="name">Name:</label>
  <input type="text" id="name" name="name" required />
  <button type="submit">Submit</button>
</form>

Note: action specifies where to send the data, and method="post" hides it from URL.

🔢 Input Types

  • text – Plain input
  • email – Email validation
  • password – Hidden characters
  • checkbox – Multiple choices
  • radio – Single choice
  • submit – Send form
<input type="email" id="email" name="email" required />
<label for="subscribe">
  <input type="checkbox" id="subscribe" name="subscribe" />
  Subscribe to newsletter
</label>

📝 Textareas and Dropdowns

<label for="bio">Bio:</label>
<textarea id="bio" name="bio" rows="4"></textarea>

<label for="country">Country:</label>
<select id="country" name="country">
  <option>Choose one...</option>
  <option value="us">USA</option>
  <option value="uk">UK</option>
</select>

🧪 Feedback Form Example

<form action="/submit" method="post">
  <label for="email">Email:</label>
  <input type="email" id="email" name="email" required />

  <label for="msg">Message:</label>
  <textarea id="msg" name="message" rows="5"></textarea>

  <button type="submit">Send Feedback</button>
</form>

📌 Summary

  • Wrap everything in <form>
  • Always use <label> + for
  • Explore input types like email, password, radio
  • Enhance with textarea and select
